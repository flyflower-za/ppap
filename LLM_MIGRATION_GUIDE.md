# LLM配置迁移指南

## 概述

PPAP平台已经统一了LLM配置系统。现在所有LLM配置都通过**系统设置**页面进行管理，废弃了之前的环境变量配置方式。

## 主要变更

### ✅ 新系统（推荐使用）

**位置**: 系统设置 → AI 模型配置

**特点**:
- 🎛️ **可视化界面**: 通过Web UI配置，无需编辑文件
- 🔐 **多模型支持**: 可以配置多个不同的LLM模型
- 🧪 **连接测试**: 内置测试连接功能，验证配置是否正确
- 🏷️ **默认模型**: 为文本和视觉任务分别设置默认模型
- 📊 **数据库存储**: 配置存储在数据库中，便于管理

### ⚠️ 旧系统（已废弃）

**位置**: `.env` 文件中的环境变量

**状态**: 仅保留用于向后兼容，不推荐使用

## 迁移步骤

### 1. 访问AI模型配置页面

登录PPAP平台后，进入：
```
系统设置 → AI 模型配置
```

### 2. 创建新的模型配置

点击"添加模型"按钮，填写以下信息：

- **名称**: 给模型起一个识别名称（如"OpenAI GPT-4"）
- **Base URL**: API端点地址
  - OpenAI: `https://api.openai.com/v1`
  - Azure: `https://your-resource.openai.azure.com/`
  - 本地模型: `http://localhost:8000/v1`
- **API Key**: 你的API密钥
- **模型名称**: 具体模型名称
  - 文本模型: `gpt-4o-mini`, `gpt-4`, `gpt-3.5-turbo`
  - 视觉模型: `gpt-4o`, `gpt-4-vision-preview`
- **模型类型**: 选择"文本"、"视觉"或"两者都支持"
- **最大Tokens**: 根据需求设置（建议2048）
- **Temperature**: 设置温度参数（建议0.1）

### 3. 测试连接

配置完成后，点击"测试连接"按钮验证配置是否正确。

### 4. 设为默认

如果这是你首选的模型，点击"设为默认"按钮。

## 配置优先级

系统按以下优先级查找LLM配置：

1. **数据库模型配置** (系统设置)
2. **数据库遗留配置** (旧版单模型配置)
3. **环境变量配置** (`.env`文件，仅向后兼容)

## 支持的提供商

新系统支持所有OpenAI兼容的API提供商：

- ✅ OpenAI (官方)
- ✅ Azure OpenAI
- ✅ Anthropic (通过代理)
- ✅ 本地模型 (Ollama, vLLM等)
- ✅ 第三方兼容服务

## 功能对比

| 功能 | 旧系统 (环境变量) | 新系统 (Web UI) |
|------|------------------|----------------|
| 多模型配置 | ❌ 不支持 | ✅ 支持 |
| 可视化管理 | ❌ 需要编辑文件 | ✅ Web界面 |
| 连接测试 | ❌ 需要手动测试 | ✅ 一键测试 |
| API Key脱敏 | ❌ 明文显示 | ✅ 自动脱敏 |
| 默认模型设置 | ❌ 不支持 | ✅ 文本/视觉分别设置 |
| 配置验证 | ❌ 无验证 | ✅ 实时验证 |

## 向后兼容性

如果你的系统中已经配置了环境变量：
- ✅ 这些配置仍然可用
- ✅ 无需立即迁移
- ⚠️ 建议逐步迁移到新系统

## 技术细节

### 配置存储位置

**新系统**:
```sql
-- 存储在数据库 Setting 表中
key = 'ai_model_profiles'
value = '[{...model profiles...}]'
```

**旧系统**:
```bash
# .env 文件
ALIYUN_ACCESS_KEY_ID=sk-...
ALIYUN_AGENT_ENDPOINT=https://api.openai.com/v1
ALIYUN_MODEL_NAME=gpt-3.5-turbo
```

### 代码调用方式

所有组件（算子、服务）现在都使用统一的配置加载函数：

```python
# 算子和服务都使用相同的配置逻辑
ai_config = await _get_ai_config(model_type="text")
```

## 常见问题

### Q: 如果我既有环境变量配置，又有Web UI配置，系统会用哪个？

A: 优先使用Web UI配置。只有当Web UI中没有配置时，才会回退到环境变量。

### Q: 我可以同时配置多个模型吗？

A: 可以！新系统支持配置多个模型，并为不同任务设置不同的默认模型。

### Q: 环境变量配置会被删除吗？

A: 不会。环境变量配置会被保留以向后兼容，但建议迁移到新系统。

### Q: 如何验证我的配置是否生效？

A: 在AI模型配置页面，点击"测试连接"按钮。

## 需要帮助？
