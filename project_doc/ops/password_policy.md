# PPAP 密码策略文档

> 本文档定义 PPAP 平台的密码要求、验证规则和安全建议。

---

## 密码要求

### 基本要求

| 要求类型 | 规则 | 强制执行位置 | 状态 |
|---------|------|-------------|------|
| **最小长度** | 至少 6 位 | - 创建用户（后端 API） | ✅ 已实现 |
| - | - 修改密码（后端） | ✅ 已实现 |
| - - 重置密码（后端） | ✅ 已实现 |
| - | - 创建用户（CLI） | ✅ 已实现 |
| - | 前端表单提示 | ✅ 已实现 |

### 复杂度要求（建议）

| 类别 | 建议 | 当前状态 |
|------|------|----------|
| **字符种类** | 建议包含大小写字母、数字、特殊字符 | ❌ 未强制 |
| **常见弱密码** | 避免使用 `123456`、`password`、`abcdef` | ❌ 未检查 |
| **与个人信息关联** | 避免使用生日、手机号、邮箱 | ❌ 未检查 |
| **密码重复** | 避免与其他网站使用相同密码 | ❌ 未检查 |

## 安全建议

### 1. 管理员账号密码

**建议要求**：
- 最小长度：**12 位**
- 必须包含：大小写字母 + 数字 + 特殊字符
- 避免使用：公司名称、`admin`、`password`

**示例**：
```
✅ 好密码：P@pp#2026!Secure
❌ 弱密码：admin123
❌ 弱密码：company@2024
```

### 2. 普通用户密码

**建议要求**：
- 最小长度：8 位
- 建议包含：字母 + 数字
- 避免使用纯数字或纯字母

**示例**：
```
✅ 好密码：user2024!
❌ 弱密码：123456
❌ 弱密码：abcdef
```

### 3. 密码重置策略

**当前机制**：
- ✅ 用户忘记密码：管理员通过 CLI 或 Web UI 重置
- ✅ 定期强制修改：未实现
- ⚠️ 密码有效期：未设置

**建议**：
- 生产环境：建议每 90-180 天提醒用户修改密码
- 高权限账号：建议每 30-60 天强制修改密码
- 泄露事件：立即强制修改所有相关密码

## 密码安全最佳实践

### 对于用户

1. **使用强密码**：
   - 混合大小写字母、数字、特殊字符
   - 长度至少 8-12 位
   - 避免个人信息

2. **不要共享密码**：
   - 不同网站使用不同密码
   - 不要通过邮件、聊天工具发送明文密码

3. **定期更换密码**：
   - 高权限账号：每 30-60 天
   - 普通账号：每 90-180 天
   - 怀疑泄露时立即更换

4. **启用多因素认证**：
   - 配置 SSO/LDAP 集成认证
   - 减少密码泄露风险

### 对于管理员

1. **初始密码设置**：
   - 避免使用默认密码
   - 通过安全渠道（面对面、企业邮箱）发送
   - 要求用户首次登录后立即修改

2. **密码重置流程**：
   - 验证用户身份（邮件、工单）
   - 生成随机临时密码
   - 要求用户立即修改

3. **离职账号处理**：
   - 员工离职时立即停用账号
   - 不要立即删除（数据可能需要保留）
   - 30 天后可彻底删除
