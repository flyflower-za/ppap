# PPAP PDF 智能校验平台 — 功能清单

> 生成日期: 2026-05-25

---

## 一、PDF 底层校验（确定性引擎）

| # | 功能 | 说明 |
|---|------|------|
| 1 | **PDF 文本层检测** | 检测是否为可检索文本型（非纯扫描件），提取页码、字符数、样本文本、图片标记、空间文本块坐标 |
| 2 | **二维码识别** | 渲染高 DPI 页面图像，OpenCV 三级策略（批量→单张→自适应增强）解码所有二维码，返回页码、内容和边界坐标 |
| 3 | **数字签名验证** | pyHanko 验证 PKCS#7 签名完整性，提取签署主体 CN/Subject/Issuer/有效期/序列号，失败时降级手动正则解析 |
| 4 | **国密签名备用检查** | 绕过 pyHanko 直接解析原始 PDF 字节，正则提取 `/Type /Sig` 签名、ByteRange、签名时间、SubFilter 算法 |
| 5 | **修订版本（增量更新）检测** | 统计 xref 交叉引用段数量判断修订次数，PyMuPDF 交叉验证，与签名数据交叉引用判断签名后是否被篡改 |
| 6 | **数字证书信息提取** | 从签名证书中提取主体、CA 机构、有效期、序列号等信息，前端折叠式 UI 展示 |

---

## 二、AI 智能校验（非确定性引擎）

| # | 功能 | 说明 |
|---|------|------|
| 7 | **文本大模型审核** | 使用 OpenAI 兼容 LLM 对 PDF 全文进行语义审核分析，支持多模型配置，未配置自动降级为 Mock |
| 8 | **视觉大模型分析** | 使用多模态 LLM 分析 PDF 页面图像（全页或裁剪区域），用于印章、公章、手写签名等视觉检测 |
| 9 | **机构自动嗅探** | 通过 LLM 识别文档签发机构（如 CTI/SGS），文本不足时自动降级为视觉分析 + 关键词 Fallback |
| 10 | **多模型 Profile 管理** | 支持多模型配置（Base URL / API Key / Model Name / Max Tokens），Key 脱敏显示，测试连接，设默认 |

---

## 三、规则引擎

| # | 功能 | 说明 |
|---|------|------|
| 11 | **动态规则引擎** | 规则解析 → 算子依赖自动解析 → 算子执行 → 规则评估，支持插件规则、LLM Prompt、Logic Graph |
| 12 | **可视化规则编辑器** | Vue Flow 拖拽式画布，节点连线构建 DAG 有向无环图，支持条件分支和变量插值 |
| 13 | **规则分类管理** | 文档分类 CRUD、关键词配置、规则按分类关联、按机构名条件触发 |
| 14 | **规则类型支持** | 关键字匹配、正则表达式、大模型 Prompt、Logic Graph 可视化流程图 |

---

## 四、文件管理

| # | 功能 | 说明 |
|---|------|------|
| 15 | **PDF 上传** | 拖拽上传、批量上传、文件预校验（后缀/MIME/≤50MB）、并发控制（≤3路） |
| 16 | **文件列表筛选** | 分页 + 多条件筛选（状态、类型、关键词、日期范围） |
| 17 | **PDF 在线预览** | pdfjs-dist 在线预览，支持二维码边界叠加高亮 |
| 18 | **文件重新校验** | 一键重新触发校验流程 |
| 19 | **文件软删除** | 单个删除 + 批量删除，不物理删除 MinIO 文件 |
| 20 | **文件自动清理** | 定时清理超期文件（Celery Beat），管理员配置保留天数和执行时间 |

---

## 五、人工审核

| # | 功能 | 说明 |
|---|------|------|
| 21 | **人工仲裁** | approve/reject 双选项，自动更新文件状态，记录审核人和时间 |
| 22 | **审核备注** | 时间线式备注系统，仅作者本人可编辑/删除 |
| 23 | **人工审核节点** | Logic Graph 中可配置人工审核节点，标记为 Human-in-the-Loop |

## 六、通知与邮件

| # | 功能 | 说明 |
|---|------|------|
| 24 | **站内通知** | 创建/查询/已读/全部已读，支持按类型过滤（success/error/warning/info） |
| 25 | **邮件通知** | 校验完成后自动发送通知邮件，支持 SMTP TLS/SSL |
| 26 | **每日汇总报告** | 每日 8:00 AM 发送校验统计邮件（总数/通过/警告/失败/Top 10 文件） |
| 27 | **邮件模板管理** | 内置模板 CRUD、预览、变量替换，支持自定义通知邮件模板 |

## 七、安全与权限

| # | 功能 | 说明 |
|---|------|------|
| 28 | **JWT 认证** | Access Token 创建/解码，密码哈希验证（bcrypt） |
| 29 | **RBAC 角色权限** | 三级角色（ADMIN / MANAGER / USER），文件/设置/操作矩阵控制 |
| 30 | **LDAP/SSO 集成** | LDAP 服务器配置、AD 组映射（PPAP-Admins/Managers/Users → 角色）、SAML 单点登录预留 |
| 31 | **审计日志** | 全链路敏感操作记录（登录、规则 CRUD、文件上传、人工仲裁），捕获 IP 地址 |
| 32 | **API Key 脱敏** | 多模型配置的 API Key 显示为 `***`，编辑时保留原值或更新 |

## 九、基础设施

| # | 功能 | 说明 |
|---|------|------|
| 35 | **PostgreSQL 15** | 异步 SQLAlchemy 引擎（asyncpg），连接池管理 |
| 36 | **Redis 7** | 缓存 + Celery broker/backend + WebSocket Pub/Sub |
| 37 | **MinIO** | S3 兼容对象存储，预签名 URL 生成，自动创建 bucket |
| 38 | **Celery 异步任务** | 后台 PDF 校验任务队列，支持进度推送 |
| 39 | **Celery Beat 调度** | 定时任务：每日 8:00 汇总报告，每日 2:00 文件清理 |
| 40 | **WebSocket 实时推送** | 校验进度实时推送至前端（Redis Pub/Sub 桥接） |
| 41 | **Nginx 反向代理** | 前端静态文件服务 + API 代理 |
| 42 | **Alembic 迁移** | 数据库结构版本管理 |
